## 项目概述

**Markdown 冒险引擎**——一个 Markdown 文件就是一个完整游戏。LLM 同时担任叙事引擎和规则裁判，通过结构化区块维持世界一致性。

### 核心特性

- **单文件架构**：所有内容在一个 `.md` 文件中，无需数据库
- **无限世界**：奇幻、科幻、末日、历史——任何你能想象的世界
- **动态叙事**：LLM 驱动的叙述者创造沉浸式故事
- **结构化状态管理**：快照为唯一真相源，防止叙事漂移
- **多层防漂移机制**：永久约束 / 速查卡锚定 / 每轮自检 / 快照自足 / 上下文刷新
- **自由探索**：没有强制主线任务，玩家决定自己的路径
- **真实后果**：行动产生持久影响，包括冒险结束
- **AI 优先设计**：模板以 AI 可直接解析为标准，最小 token 成本

## 快速开始

### 前置要求

- 现代 LLM（DeepSeek、GPT-4、Claude 等）
- 无需安装任何软件

### 开始游戏

1. **复制模板**：打开 `markdown-game-template.md`（中文）或 `markdown-game-template-en.md`（英文），复制全部内容
2. **粘贴到 LLM 对话框**：将模板粘贴到 DeepSeek、GPT、Claude 等任意 LLM 的对话窗口
3. **回答两个问题**：LLM 会询问你——
   - "你想要什么样的世界？"（奇幻、科幻、末日、历史等）
   - "你是谁？"（姓名、种族、职业、背景）
4. **开始冒险**：LLM 将生成世界并开始叙事

### 示例会话

```
玩家：我想要一个有特殊能力和龙的中世纪奇幻世界。
玩家：我是 Elara，一个寻找古代遗物的半精灵游侠。

叙述者：[生成世界设定、势力和起始场景]
叙述者：你站在低语之森的边缘...
```

### 模板分区

| 分区 | 用途 | 更新频率 |
|------|------|----------|
| 永久约束 | 最高优先级规则，不可覆盖 | 永不 |
| 速查卡 | 超压缩核心规则，每次回复前必读 | 极少 |
| Snapshot | 唯一真相源（自足化设计） | 每次交互后 |
| 开始流程 | 初始化/构建世界/角色模板 | 初始化时 |
| 游戏流程 | 自检/输出模块/命令列表 | 固定 |
| 游戏规则 | 判定/遭遇/生存/成长 | 固定 |

## 设计哲学

1. **简洁性**：无需安装，只需 Markdown
2. **便携性**：GitHub、本地、云存储随处可用
3. **透明性**：完整游戏状态可见且可编辑
4. **防漂移**：多层防线（永久约束/速查卡/每轮自检/快照自足/上下文刷新），防止 LLM 记忆漂移
5. **上下文管理**：结构化分区 + 里程碑压缩 + 主动刷新，长会话保持一致

## 防漂移机制

长会话中 LLM 丢失上下文、忘记规则是最大痛点。本模板引入多层防漂移机制：

| 层级 | 技术 | 原理 |
|------|------|------|
| 第1层 | 永久约束表格 | 声明"必须/不可"约束，模型视为最高优先级 |
| 第2层 | 速查卡锚定 | 结构化表格格式，模型对表格敏感度高于纯文本 |
| 第3层 | 每轮自检输出 | 回复前强制8项自检，防止进入冷数据区 |
| 第4层 | 快照自足化 | Snapshot 包含完整游戏状态，仅凭此块可恢复游戏 |
| 第5层 | 上下文刷新 | 每10回合主动压缩 + 玩家可手动"刷新"/"校准" |

### 玩家操作

当察觉 LLM 开始"胡言乱语"时，输入以下任一指令：
- `刷新` / `refresh` / `校准`

叙述者将执行刷新流程：重写规则到最新位置 → 压缩历史 → 输出矫正后的完整快照。

## 游戏机制

### 回合系统
- 有效世界内行动 → 回合+1；闲聊/查询/澄清不消耗
- 章节切换不额外+1

### 行动判定
- 确定性（走/拾/说）→ 直接成功
- 风险性（攀/渡/说服）→ 五档判定（大成功/成功/勉强/失利/严重受挫）
- 判定矩阵：属性值 vs 难度等级（3/5/7/9），装备+1~2，情境-1~2

### 遭遇系统
- 回合制：玩家→判定→情境响应→循环
- 消耗：按行动类型和属性决定，大成功消耗减半
- 防御：闪避(敏捷) / 招架(体质) / 护具减轻损伤
- 挑战3级：普通→危险→严峻
- 不良状态：疲劳/困惑/失手/踉跄

### 生存系统
- 耐力：体质×2，耗尽=冒险结束，休息或物品可恢复
- 精力：100→每回合-1(紧张额外-1)，精力低→属性减益，耗尽→耐力-2/回合
- 经济：货币，买标价/卖半价，议价需魅力判定，不可为负

### NPC系统
- NPC有自主行为逻辑
- 世界持续运转，消息随时间传播
- 态度随玩家行为动态变化

### 成长
- 触发：克服重大挑战/完成委托/发现真相/探索新区/每10回合
- 幅度：属性+1 或 1新技能，每次只选一项
- 7个预设技能：招架/追踪/草药/解密/驯兽/洞察/烹饪

## 高级用法

### 快照云端同步

游戏状态（快照）可同步至云端，实现跨会话持久化和多端恢复。支持任意 REST API 端点，在模板头部 yaml 中配置 `snapshot_store` 和认证信息即可。

**可选存储方案：**

| 方案 | 优点 | 缺点 |
|------|------|------|
| Supabase | 支持查询/历史 | 需建表 |
| GitHub Gist | 版本历史 | 需 token |
| CloudBase | 微信生态 | 需登录 |

### 上下文管理
长会话防溢出：
1. 日志自动压缩（每10回合）
2. 场景切换时精简已探索区域
3. LLM 优先读取速查卡 + Snapshot + 当前场景 + 世界状态
4. 每10回合主动执行上下文刷新（规则重写到最新位置）

### 里程碑压缩（每10回合）
- 归档快照到日志
- 压缩旧日志为结构化摘要（关键事件/重要变化/目标线索）
- 精简世界状态和已探索区域
- 重写速查卡和快照到最新位置
- 通知玩家

### 自定义选项
- 多人模式：添加多个角色卡
- 模组系统：世界设定拆分为独立 `.md` 文件
- 难度调整：修改难度阈值和判定矩阵
- 自定义机制：制作、关系、声望等
