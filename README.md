## 项目概述

**Markdown 冒险引擎**——一个 Markdown 文件就是一个完整游戏。LLM 同时担任叙事引擎和规则裁判，通过结构化区块维持世界一致性。

### 核心特性

- **单文件架构**：所有内容在一个 `.md` 文件中，无需数据库
- **无限世界**：奇幻、科幻、末日、历史——任何你能想象的世界
- **动态叙事**：LLM 驱动的地下城主创造沉浸式故事
- **结构化状态管理**：快照为唯一真相源，防止叙事漂移
- **自由探索**：没有强制主线任务，玩家决定自己的路径
- **真实后果**：行动产生持久影响，包括永久死亡
- **AI 优先设计**：模板以 AI 可直接解析为标准，最小 token 成本

## 快速开始

### 前置要求

- 现代 LLM（GPT-4、Claude 或类似模型）
- Markdown 编辑器（VS Code、Obsidian 等）

### 开始游戏

1. **复制模板**：使用 `markdown-game-template.md` 作为起点
2. **用 LLM 初始化**：分享模板并回答两个问题：
   - "你想要什么样的世界？"（奇幻、科幻、历史等）
   - "你是谁？"（姓名、种族、背景）
3. **开始冒险**：LLM 将生成世界并开始冒险

### 示例会话

```
玩家：我想要一个有魔法和龙的中世纪奇幻世界。
玩家：我是 Elara，一个寻找古代遗物的半精灵游侠。

DM：[生成世界设定、势力和起始场景]
DM：你站在低语之森的边缘...
```

## 项目结构

```
md-adventure/
├── README.md                    # 本文件
└── markdown-game-template.md    # 完整游戏模板
```

### 模板分区

| 分区 | 用途 | 更新频率 |
|------|------|----------|
| 快照 | 唯一真相源（HP/饥饿/状态/近期事件） | 每次交互后 |
| 系统协议 | LLM 行为约束（禁令/细描/矫正） | 极少 |
| 世界设定 | 背景、势力、地理、天气、终局 | 偶尔 |
| 游戏规则 | 判定/战斗/饥饿/经济/时间/成长 | 固定 |
| 角色 | 属性、物品、金币、技能 | 变动时 |
| 当前场景 | 位置、时间、氛围、感知 | 每回合 |
| 历史日志 | 倒序事件流 | 每回合 |
| 世界状态 | NPC、势力动态、已探索/未探索 | 关键事件后 |

## 设计哲学

1. **简洁性**：无需安装，只需 Markdown
2. **便携性**：GitHub、本地、云存储随处可用
3. **透明性**：完整游戏状态可见且可编辑
4. **防漂移**：快照为唯一数据源，矫正机制防止 LLM 记忆漂移
5. **上下文管理**：结构化分区 + 里程碑压缩，长会话保持一致

## 游戏机制

### 回合系统
- 有效世界内行动 → turn+1；闲聊/查询/澄清不消耗
- 章节切换可展现完整数据，不额外+1

### 行动判定
- 确定性（走/拾/说）→ 直接成功
- 风险性（战/攀/说服）→ 五档判定（大成功/成功/勉强/失败/大失败）
- 判定矩阵：属性值 vs 难度等级（DC3/5/7/9），装备+1~2，情境-1~2

### 战斗系统
- 回合制：玩家→判定→敌方→循环
- 伤害：1+属性÷2，大成功×1.5
- 防御：闪避(敏捷)/格挡(体质)/护甲固定减伤(1-4)
- 敌人4级：杂兵→精锐→精英→Boss
- 战斗状态：流血/眩晕/缴械/倒地

### 生存系统
- HP：体质×2，归零=永久死亡，休息+1/回合
- 饥饿：100→每回合-1，≤30/-1属性，≤10/-2属性+移动受限，0→HP-2/回合
- 经济：GP货币，议价需魅力判定
- 时间：行动推进0.5-4h，8时段昼夜循环，夜间视野递减
- 天气：9种天气影响视野/饥饿/战斗/特殊效果

### NPC系统
- NPC有自主活动时段和行为逻辑
- 世界持续运转，消息随时间传播
- 态度随玩家行为动态变化

### 成长
- 触发：击败强敌/完成委托/发现真相/存活每20回合
- 幅度：属性+1 或 1新技能，每次只选一项
- 7个预设技能：格挡/追踪/草药/开锁/驯兽/洞察/烹饪

## 高级用法

### 上下文管理
长会话防溢出：
1. 日志自动压缩（每20回合）
2. 场景切换时精简已探索区域
3. LLM 优先读取快照+当前场景+世界状态

### 里程碑压缩（每20回合）
- 归档快照到日志
- 压缩旧日志为结构化摘要（关键事件/重要变化/目标线索）
- 精简世界状态和已探索区域
- 通知玩家

### 自定义选项
- 多人模式：添加多个角色卡
- 模组系统：世界设定拆分为独立 `.md` 文件
- 难度调整：修改 DC 阈值和判定矩阵
- 自定义机制：制作、关系、声望等
