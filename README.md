## 项目概述

**Markdown 冒险引擎**——一个 Markdown 文件就是一个完整游戏。LLM 同时担任叙事引擎和规则裁判，通过结构化区块维持世界一致性。

### 核心特性

- **单文件架构**：所有内容在一个 `.md` 文件中，无需数据库
- **无限世界**：奇幻、科幻、末日、历史——任何你能想象的世界
- **动态叙事**：LLM 驱动的叙述者创造沉浸式故事
- **结构化状态管理**：快照为唯一真相源，防止叙事漂移
- **五层防漂移机制**：元指令锁定 / XML标签锚定 / 每轮自检输出 / 快照自足 / 上下文刷新
- **自由探索**：没有强制主线任务，玩家决定自己的路径
- **真实后果**：行动产生持久影响，包括冒险终止
- **AI 优先设计**：模板以 AI 可直接解析为标准，最小 token 成本

## 快速开始

### 前置要求

- 现代 LLM（DeepSeek、豆包、GPT-4、Claude 等）
- 无需安装任何软件

### 开始游戏

1. **复制模板**：打开 `markdown-game-template.md`（中文）或 `markdown-game-template-en.md`（英文），复制全部内容
2. **粘贴到 LLM 对话框**：将模板粘贴到 DeepSeek、豆包、GPT、Claude 等任意 LLM 的对话窗口
3. **回答两个问题**：LLM 会询问你——
   - "你想要什么样的世界？"（奇幻、科幻、末日、历史等）
   - "你是谁？"（姓名、种族、职业、背景）
4. **开始冒险**：LLM 将生成世界并开始叙事

### 示例会话

```
玩家：我想要一个有魔法和龙的中世纪奇幻世界。
玩家：我是 Elara，一个寻找古代遗物的半精灵游侠。

叙述者：[生成世界设定、势力和起始场景]
叙述者：你站在低语之森的边缘...
```

### 模板分区

| 分区 | 用途 | 更新频率 |
|------|------|----------|
| 永久约束 `<Permanent_Constraints>` | 元指令锁定，不可覆盖的最高优先级规则 | 永不 |
| 规则速查卡 `<Rule_Quick_Ref>` | 超压缩核心规则，每次回复前必读 | 极少 |
| 核心禁令 `<Core_Bans>` | 6条不可违反的行为红线 | 永不 |
| 系统协议 | LLM 行为约束（细描/矫正/自检） | 极少 |
| 世界设定 | 背景、势力、地理、终局 | 偶尔 |
| 游戏规则 | 判定/遭遇/精力/经济/成长 | 固定 |
| 角色 | 属性、物品、金币、技能 | 变动时 |
| 快照 `<Snapshot>` | 唯一真相源（自足化设计），可同步至云端 | 每次交互后 |
| 当前场景 | 位置、氛围、感知 | 每回合 |
| 历史日志 | 倒序事件流 | 每回合 |
| 世界状态 | NPC、势力动态、已探索/未探索 | 关键事件后 |

## 设计哲学

1. **简洁性**：无需安装，只需 Markdown
2. **便携性**：GitHub、本地、云存储随处可用
3. **透明性**：完整游戏状态可见且可编辑
4. **防漂移**：五层防线（元指令/XML标签/每轮自检/快照自足/上下文刷新），防止 LLM 记忆漂移
5. **上下文管理**：结构化分区 + 里程碑压缩 + 主动刷新，长会话保持一致

## 防漂移机制（v1.1 核心升级）

长会话中 LLM 丢失上下文、忘记规则是最大痛点。v1.1 引入五层防漂移机制：

| 层级 | 技术 | 实现 | 原理 |
|------|------|------|------|
| 第1层 | 元指令锁定 | `<Permanent_Constraints>` 标签包裹6条永久约束 | 声明"无论对话多久始终激活"，模型将其视为最高优先级 |
| 第2层 | XML标签锚定 | `<Rule_Quick_Ref>` / `<Core_Bans>` / `<Snapshot>` | 模型对结构化标签敏感度高于纯文本，标签便于指代和引用 |
| 第3层 | 每轮自检输出 | 回复末尾强制输出「规则检查清单」 | 强制模型每轮重新"读取"规则，防止其进入冷数据区 |
| 第4层 | 快照自足化 | `<Snapshot>` 包含属性/物品/金币/技能/目标/难度 | 仅凭快照即可恢复游戏，不依赖历史对话 |
| 第5层 | 上下文刷新 | 每10回合主动压缩+玩家可手动"刷新"/"校准" | 将规则从"远期记忆"重写为"近期记忆"，等效重置上下文 |

### 玩家操作

当察觉 LLM 开始"胡言乱语"时，输入以下任一指令：
- `刷新` / `refresh` / `校准`

叙述者将执行刷新流程：重写规则到最新位置 → 压缩历史 → 输出矫正后的完整快照。

## 游戏机制

### 回合系统
- 有效世界内行动 → turn+1；闲聊/查询/澄清不消耗
- 章节切换可展现完整数据，不额外+1

### 行动判定
- 确定性（走/拾/说）→ 直接成功
- 风险性（攀/渡/说服）→ 五档判定（大成功/成功/勉强/失败/大失败）
- 判定矩阵：属性值 vs 难度等级（3/5/7/9），装备+1~2，情境-1~2

### 遭遇系统
- 回合制：玩家→判定→情境响应→循环
- 消耗：1+属性÷2，大成功×1.5
- 抵抗：闪避(敏捷)/格挡(体质)/防护固定减耗(1-4)
- 挑战4级：轻微→中等→重大→极限
- 负面态：疲劳/困惑/失手/踉跄

### 生存系统
- 耐力：体质×2，归零=冒险终止，休息+1/回合
- 精力：100→每回合-1，≤30/-1属性，≤10/-2属性+移动受限，0→耐力-2/回合
- 经济：GP货币，议价需魅力判定

### NPC系统
- NPC有自主行为逻辑
- 世界持续运转，消息随时间传播
- 态度随玩家行为动态变化

### 成长
- 触发：克服重大挑战/完成委托/发现真相/探索新区/每10回合
- 幅度：属性+1 或 1新技能，每次只选一项
- 7个预设技能：格挡/追踪/草药/开锁/驯兽/洞察/烹饪

## 高级用法

### 快照云端同步

游戏状态（快照）可同步至云端，实现跨会话持久化和多端恢复。支持任意 REST API 端点，在模板头部 yaml 中配置 `snapshot_store` 和认证信息即可。

**可选存储方案：**

| 方案 | 优点 | 缺点 |
|------|------|------|
| Supabase | 支持查询/历史 | 需建表 |
| GitHub Gist | 版本历史 | 需 token |
| CloudBase | 微信生态 | 需登录 |

### 上下文管理
长会话防溢出：
1. 日志自动压缩（每10回合）
2. 场景切换时精简已探索区域
3. LLM 优先读取 `<Rule_Quick_Ref>` + `<Snapshot>` + 当前场景 + 世界状态
4. 每10回合主动执行上下文刷新（规则重写到最新位置）

### 里程碑压缩（每10回合）
- 归档快照到日志
- 压缩旧日志为结构化摘要（关键事件/重要变化/目标线索）
- 精简世界状态和已探索区域
- 重写 `<Rule_Quick_Ref>` 和 `<Snapshot>` 到最新位置
- 通知玩家

### 自定义选项
- 多人模式：添加多个角色卡
- 模组系统：世界设定拆分为独立 `.md` 文件
- 难度调整：修改难度阈值和判定矩阵
- 自定义机制：制作、关系、声望等
