## 🎮 项目概述

**Markdown 冒险引擎**是一个创新的游戏框架，其中**一个 Markdown 文件就是一个完整的游戏**。大型语言模型（LLM）同时担任叙事引擎和规则裁判，通过结构化区块维持世界一致性。

### 核心特性

- ✨ **单文件架构**：所有内容都在一个 `.md` 文件中——无需数据库，无需复杂设置
- 🌍 **无限可能**：奇幻、科幻、末日、历史——任何你能想象的世界
- 🎲 **动态叙事**：LLM 驱动的地下城主创造沉浸式、响应式的故事
- 📊 **结构化状态管理**：清晰分离世界状态、角色数据和游戏规则
- 🔀 **自由探索**：没有强制主线任务——你决定自己的路径和目标
- ⚖️ **真实后果**：行动会产生持久影响，包括永久死亡
- 📝 **人类和 AI 可读**：易于阅读、修改和扩展

## 🚀 快速开始

### 前置要求

- 现代 LLM（GPT-4、Claude 或类似模型）
- Markdown 编辑器（VS Code、Obsidian 等）

### 开始游戏

1. **复制模板**：使用 `markdown-game-template.md` 作为起点
2. **用 LLM 初始化**：分享模板并回答两个问题：
   - "你想要什么样的世界？"（奇幻、科幻、历史等）
   - "你是谁？"（姓名、种族、背景）
3. **开始冒险**：LLM 将生成世界并开始冒险

### 示例会话

```
玩家：我想要一个有魔法和龙的中世纪奇幻世界。
玩家：我是 Elara，一个寻找古代遗物的半精灵游侠。

DM：[生成世界设定、势力和起始场景]
DM：你站在低语之森的边缘...
```

```
game/

```

### 模板分区

| 分区 | 用途 | 更新频率 |
|------|------|----------|
| ⚡ 当前快照 | 当前回合的最小真相源 | 每次交互后 |
| 📋 系统协议 | LLM 行为指令（隐藏规则） | 极少 |
| 🌍 世界设定 | 背景、势力、地理、魔法系统 | 偶尔 |
| 📜 游戏规则 | 核心机制、判定逻辑、边界 | 固定 |
| 🧑 角色 | 玩家属性、物品、状态 | 每回合 |
| 📍 当前场景 | 位置描述和即时情境 | 每回合 |
| 📖 历史日志 | 按回合记录的事件流 | 每回合 |
| 🗺️ 世界状态 | NPC 状态、势力变化、已触发事件 | 关键事件后 |

## 🎯 设计哲学

### 为什么采用这种方法？

1. **简洁性**：无需安装，无需依赖，只需 Markdown
2. **便携性**：随处可用——GitHub、本地文件、云存储
3. **透明性**：完整游戏状态可见且可编辑
4. **灵活性**：易于自定义规则、添加功能或创建变体
5. **上下文管理**：结构化分区帮助 LLM 在长时间会话中保持一致性

### 核心原则

- **沉浸优先**：第二人称叙事让玩家保持投入
- **玩家自主权**：没有强制路径——自由探索
- **真实后果**：真实的利害关系，包括永久死亡
- **状态一致性**：结构化数据防止叙事漂移
- **生存压力**：饥饿系统增加紧迫感而不至于压倒性

## 🛠️ 高级用法

### 上下文管理

对于 extended 游戏会话：

1. **日志压缩**：将旧回合压缩为摘要（每 20 回合自动执行）
2. **场景简化**：离开时将已探索区域总结为摘要
3. **选择性读取**：LLM 专注于角色 + 当前场景 + 世界状态

### 自定义选项

- **多人模式**：添加多个角色卡
- **模组系统**：将世界设定拆分为独立的 `.md` 文件
- **难度调整**：修改游戏规则部分的判定规则
- **自定义机制**：添加新系统（制作、关系等）

### 里程碑压缩

每 20 回合，系统自动：
- 归档当前快照
- 压缩历史日志
- 简化世界状态
- 通知玩家

这可以防止上下文溢出，同时保留必要信息。

## 📖 游戏机制

### 回合系统

- **回合定义**：仅在执行有效的世界内行动时递增
- **非回合操作**：提问、状态查询、规则澄清不消耗回合
- **章节机制**：新地点或重大事件开启新章节

### 行动判定

- **确定性行动**：走路、拾取物品——总是成功
- **风险性行动**：战斗、说服——基于属性和情境评估
- **五档结果**：大成功 / 成功 / 勉强 / 失败 / 大失败

### 生存系统

- **饥饿度**：每回合递减（基础 -1，剧烈活动额外 -1）
- **效果**：低饥饿度降低力量和体质
- **死亡**：饥饿度归零导致持续生命值损失

### 死亡与永久性

- 生命值归零 = 永久死亡
- 无复活机制
- 可选择用新角色或新世界重新开始
